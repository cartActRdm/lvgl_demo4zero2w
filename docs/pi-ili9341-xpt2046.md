# Raspberry Pi Zero 2 W + ILI9341 + XPT2046

这份说明现在对应的是 **userspace 直驱显示 + 触摸** 路线：

- 显示：`spidev` + `libgpiod` + LVGL flush
- 触摸：`spidev` + 可选 `libgpiod` IRQ + LVGL indev

## 已知硬件参数

根据已经跑通的 Python 直驱验证与这次目标接线：

- 显示 SPI: `spi0.0` / `/dev/spidev0.0`
- 触摸 SPI: `spi0.1` / `/dev/spidev0.1`
- 显示 CE: CE0
- 触摸 CE: CE1
- DC: GPIO24
- RESET: GPIO25
- 触摸 IRQ: GPIO23（PENIRQ，低有效）
- 宽高: `240x320`
- 旋转：`0` 或 `180` 可用
- `270` 失败，不建议默认使用

## 现在 C 后端做了什么

项目里的 `src/pi_ili9341.c` 会：

1. 打开 `/dev/spidev0.0`
2. 用 `libgpiod` 申请 DC / RESET 两根 GPIO
3. 发送与 Python 验证版一致的 ILI9341 初始化序列
4. 把 LVGL 的 RGB565 刷新区域直接写入 LCD

项目里的 `src/pi_xpt2046.c` 会：

1. 打开 `/dev/spidev0.1`
2. 可选地用 `libgpiod` 读取 GPIO23 PENIRQ（低电平表示按下）
3. 读取 X/Y/Z 原始值
4. 对 X/Y 做 5 次采样取中位值
5. 按环境变量给的原始范围映射到屏幕坐标
6. 注册成 LVGL pointer input 回调

如果 IRQ 线拿不到，它会退回到纯 SPI 压力轮询，不至于直接失效。

## Pi 端准备

### 1. 启用 SPI

```bash
sudo raspi-config
# Interface Options -> SPI -> Enable
sudo reboot
```

### 2. 确认设备

```bash
ls -l /dev/spidev*
ls -l /dev/gpiochip*
```

应该至少能看到：

- `/dev/spidev0.0`
- `/dev/spidev0.1`
- `/dev/gpiochip0`

### 3. 安装依赖

```bash
cd ~/lvgl-zero2w
./scripts/bootstrap-pi-trixie.sh
./scripts/fetch-third-party.sh
```

### 4. 编译

```bash
./scripts/configure-pi-native.sh
./scripts/build-pi.sh
```

### 5. 运行

```bash
./scripts/run-pi.sh
```

## 触摸相关调参

默认值：

```bash
XPT2046_SPI_DEV=/dev/spidev0.1
XPT2046_IRQ_GPIO=23
XPT2046_SPI_SPEED=2000000
XPT2046_SPI_MODE=0
XPT2046_RAW_X_MIN=200
XPT2046_RAW_X_MAX=3900
XPT2046_RAW_Y_MIN=200
XPT2046_RAW_Y_MAX=3900
XPT2046_SWAP_XY=1
XPT2046_INVERT_X=0
XPT2046_INVERT_Y=0
XPT2046_PRESSURE_MIN=80
```

### 如果触摸方向不对

先试：

```bash
XPT2046_SWAP_XY=0 ./scripts/run-pi.sh
XPT2046_INVERT_X=1 ./scripts/run-pi.sh
XPT2046_INVERT_Y=1 ./scripts/run-pi.sh
```

### 如果边界不准

调原始范围：

```bash
XPT2046_RAW_X_MIN=300 XPT2046_RAW_X_MAX=3800 ./scripts/run-pi.sh
XPT2046_RAW_Y_MIN=250 XPT2046_RAW_Y_MAX=3850 ./scripts/run-pi.sh
```

### 如果 IRQ 有问题

禁用 IRQ，只靠压力轮询：

```bash
XPT2046_IRQ_GPIO=-1 ./scripts/run-pi.sh
```

### 5. 原始坐标 / 四角校准

直接打印原始触摸值：

```bash
./scripts/run-pi.sh --touch-raw
```

在屏幕上跑四角校准并输出建议环境变量：

```bash
./scripts/run-pi.sh --touch-calibrate
```

也可以用环境变量切模式：

```bash
LVGL_PI_MODE=touch-calibrate ./scripts/run-pi.sh
```

校准结束后会打印建议的：

- `XPT2046_RAW_X_MIN/MAX`
- `XPT2046_RAW_Y_MIN/MAX`
- `XPT2046_SWAP_XY`
- `XPT2046_INVERT_X`
- `XPT2046_INVERT_Y`

## 为什么现在值得直接做 touch userspace

你这块屏已经用 Python 直驱证明了：

- SPI 设备是通的
- DC / RESET 控制是通的
- ILI9341 初始化序列是可行的

同样地，XPT2046 也是个很适合 userspace 直接读的 SPI 触摸控制器。
对当前这个项目目标来说，直接把它挂成 LVGL pointer，路径更短，也更方便在接线和校准还没完全定型时快速试错。

## 已知限制

- 默认按 `240x320` 竖屏处理
- 当前优先适配 Pi OS Trixie 64-bit
- 触摸校准是“先能用”的默认值，不保证你的板子开箱即精准
- 没有背光 GPIO 控制
- 没做 DMA 优化
- 没在这个仓库里加入 270 旋转修复，因为你当前实测它就是失败的
